|TCP/IP模型 -> 将OSI模型的应用层, 表现层和会话层统称为应用层; 数据链路层和物理层统称为接口层

\OSI网络模型
    |OSI模型最大的贡献 -> 服务, 接口, 协议

    |应用层 -> 网络服务接口. 如DNS, HTTP, SMTP, FTP, Telnet等
    |表现层 -> 数据的格式化转换. 如压缩和解压缩, 加密和解密
    |会话层 -> 访问验证, 会话管理. 如服务器端用户登录验证
    |传输层 -> 为上层提供透明可靠的端到端数据传输服务. 如TCP, UDP
    |网络层 -> 地址解析, 路由, 网际互联, 控制和传递消息. 如IP, ARP, RARP,ICMP
    |数据链路层 -> 操作并传输数据帧. 采用差错控制和流量控制保证数据的正确顺序, 无差错和完整性
    |物理层 -> 传输比特流(直接的数据传输)

\物理层
    |编码 -> 用数字信号传输数据
    |调制 -> 用模拟信号传输数据
    |比特率 -> 信道中二进制码每秒的传输速率(比特数)
    |波特率 -> 调制速率. 即信号波形的变换频率(或码元传输速率) 波特率 = 比特率/单个码元的比特数
    |数据传输速率 -> 香农定理: C = B * lg(1 + S/N) C: 最大传输速率, B: 带宽, S/N: 信噪比

    |电路交换 -> 线路独占, 一次性传输报文
    |报文交换 -> 线路非独占, 一次性传输报文
    |分组交换 -> 线路非独占, 分组传输报文

    |信道复用(共享)技术
        |频分复用(FDM) -> 所有用户在同一时间占用不同的频率带宽. 常用于有线电视信号传输
        |时分复用(TDM) -> 所有用户在不同时间占用同一频率带宽. 将数据划分成等长的时分复用帧
        |波分复用(WDM) -> 使用一根光纤同时传输多个频率接近的光载波信号(光的频分复用)
        |码分多址复用(CDMA) -> 所有用户在同一时间使用相同的频率带宽(各用户使用互不重叠的码型)
            -> 有很强的抗干扰能力. 用于移动通信(无线局域网)

\数据链路层
    |以太网 -> 基带总线局域网(基本等同于局域网). 目前最常用的是IEEE 802.3标准以太网
        -> 以太网提供的服务是无连接(无需建立连接即可传输数据)的不可靠(尽最大努力)的交付
        -> 适配器对发送的数据帧不进行编号, 也不要求对方返回确认
    |MAC地址 -> 物理层和数据链路层使用的地址(48位). 计算机硬件地址. 固化在适配器(网卡)的ROM中
    |通信适配器 -> 网卡. 在计算机(CPU, 存储器)和为外部局域网之间传输数据帧
        -> 适配器和计算机之间并行传输(主板上的IO总线);适配器和局域网之间串行传输(电缆或双绞线)
        -> 由于IO总线和局域网的数据传输速率不一样, 因此适配器中有一个存储芯片对数据进行缓存
    |曼彻斯特编码 -> 以码元中间电平跳变为数据. 由高到低为0, 由低到高为1
        -> 以太网发送的数据都采用曼彻斯特编码
    |CSMA/CD -> 载波监听多点接入/碰撞检测协议. 以太网半双工传输协议, 对全双工传输的以太网无效
        -> 这里的载波不是频分复用FTM的载波. 没有使用频分复用
        |端对端传输时间*2 -> 碰撞确认等待时间(争用期)
            |最短距离情形 -> Data/帧传输速率 = 端对端传输时间*2 = (两端距离/信号传输速率)*2

    -> 差错检测的目的是保证向上层提交的数据帧中, 有差错的都已经被丢弃了

    |网桥 -> 根据MAC帧的目的地址对收到的帧进行转发和过滤. 连接局域网, 扩展以太网
        |MAC帧 -> 源地址和目的地址(都使用硬件地址)
        |转发帧 -> 网桥通过帧的源地址建立转发表. 转发表无记录则向接收端口外的其他端口广播帧信息
        |丢弃帧 -> 网桥上帧的接收端口和目的端口相同或客户端地址和帧的目的地址不一致
        |生成树算法 -> 没有回路的网络拓扑

    |10BASE-T -> 10MB/s的传输速率的双绞线以太网.BASE指传输信号为基带信号,T指传输媒介为双绞线
    |100BASE-T -> 快速以太网. 可用全双工传输而不会发生碰撞, 也可选择半双工传输
    |1000BASE-T|X -> 吉比特(千兆)以太网. 可选择全双工和半双工两种传输模式
     -> 以上以太网的帧格式完全相同

\网络层
    |IP -> 网络层主要协议. 提供分组交换服务
        -> IP地址是网络层及以上各层使用的地址. 用软件实现. 保存于存储器中
    |ARP -> 将IP地址转换为MAC地址
    |RARP -> 将MAC地址转换为IP地址
    |ICMP -> 控制和传递消息

\应用层
    |DNS -> 使用UDP用户数据报传输数据(为了减少开销)

    |电子邮件
        |SMTP -> 邮件发送协议. 只能传输7位ASCII码文本
        |POP3 -> 邮件读取协议
        |用户代理 -> 电子邮件客户端

        -> 用户代理向电子邮件服务器发送邮件使用SMTP协议; 接收用户代理读取邮件使用POP3协议
        -> SMTP和POP3都是基于TCP连接工作的
        -> SMTP不使用中间邮件服务器, TCP连接总是在发送方和接收方的邮件服务器之间直接建立

    |FTP -> 基于TCP连接
        |主进程 -> 接收新的请求
        |从属进程 -> 主进程为每一个新请求分配从属进程进行处理(主从并发)
            |控制进程 -> 通过控制连接接收客户端发过来的传送请求. 控制连接整个会话期间保持打开
            |数据传输进程 -> 控制进程收到传送请求后创建数据进程和数据连接进行数据传输.传完释放
        |端口号 -> 服务器端: 控制连接21, 数据连接20; 客户端自定义

    |HTTP -> 基于TCP连接
