-> 基于Apache-JDK的分布式集群，用于大数据高速运算和存储
|应用场景 -> 搜索，日志处理，推荐系统，数据分析，视频图像分析，数据保存

1.HDFS(Hadoop Distributed File System 分布式文件系统)模块 -> 高容错性的廉价分块存储系统
    1.流式数据访问 -> 来一点处理一点(二进制Byte流)，不必等待整块加载完毕(会造成很大的网络延迟和内存消耗)，将计算好的结果追加到文件末尾
    2.分块存储 -> 将一个文件分块(Block), 通常每块大小为64M。HDFS将一个大文件平均分块后存储到不同计算机上，其意义在于读取原文件时可同时从不同主机并行读取，效率很高
    3.廉价硬件 -> 用几十台廉价计算机就能搭建起一个大数据集群
    4.高容错性 -> 将同一文件块副本存储于多个主机，如果其中一台宕机，不会影响对该文件块的读取
    5.超大文件

    |NameNode -> 保存整个文件系统的目录，分块和文件信息。由唯一一台主机专门保存，主机出错，NameNode失效。Hadoop2.x开始支持activity-standy模式——主机失效，启用备用机运行NameNode
    |DataNode -> 分布在廉价计算机集群上，用于存储Block块文件

2.YARN模块 -> 通用资源协同和任务调度框架。可运行MapReduce，Spark，Storm等框架

3.Map-Reduce模块 -> 分布式的并行计算框架
